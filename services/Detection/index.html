What of TERAWIN?

Disruptive, FAST hardware technique for Detecting:

  • Peak Amplitude in a RADAR Range Cell/Bin;
  • Time from the beginning of the Range Cell/Bin to this Peak Amplitude;
  • Number of times the Constant False Alarm Rate (CFAR) hardware-Threshold has been exceeded in the Range Cell/Bin (False Alarm Rate); and
  • Conditions to be reported by optimized Vectored Interrupt

The difference in Time between two RADAR Pulses, provides a Super-Accurate (far superior to DOPPLER) measure of the Target/Obstructions’ movement towards/away from the RADAR.

The Peak Amplitude, Time and CFAR circuit is preceded by selectable hardware Pulse Integration, Pulse Canceller; and Universal Filter (DUF) that can be Programmed on a Range Cell/Bin basis as a:

  • Low Pass Filter
  • High Pass Filter
  • Band Pass Filter
  • Notch Filter

This UF performs the equivalent of a DOPPLER Filter Bank in Pulse RADAR, and FFT transformation of Time into Frequency for CW RADAR. It also allows Targets/Obstructions that are not of interest, to be filtered out very early in Receiver Processing Chain; saving on Storage and Processing.

Why of TERAWIN?

  • SWaPC + Safety
  • Patented DynamicRangeTM adjusts each Range Cell/Bin size so that there is only one Target/Obstruction in the Range Cell/Bin. This allows the TERAWIN Hardware Process to produce an unambiguous Peak Amplitude and Time to Peak Amplitude. It also allows the RADAR to be used for Long Range and Short Range (a potential saving of 1 RADAR).
  • Hardware Technique (=> FAST RESPONSE – Think Emergency Braking) that can be implemented in very low cost Medium Scale Integration (MSI) chip or a very small area of a System On a Chip (SOC):
    • Dynamic Filtering on a Range Cell/Bin basis (Pulse Integration, Pulse Canceller, with Low Pass, High Pass, Band Pass and Notch Filter). Equivalent to DOPPLER Filter Bank in Pulse RADAR. Equivalent of Time-To-Frequency FFT in FMCW RADAR.
    • Peak Amplitude Detection in the Range Cell/Bin. Equivalent to Target Detection in Pulse and FMCW.
    • Time (Clock Count) into the Range Cell/Bin of this Peak Amplitude.
      • (2 pulses provide 2 Clock Counts. If Counts the same, Target/Obstruction is stationary. If Counts are different, Target/Obstruction is moving. Equivalent to DOPPLER in both Pulse and FMCW.
      • Clock frequency determines distance-accuracy in Range and Radial movement. For example, a 30 GHz clock will show distance accuracy to 1 cm.
      • Much simpler and much more accurate than DOPPLER).
    • Constant False Alarm Rate (CFAR) Detection and Thresholding (hardware Threshold set by software).
    • Peak Amplitude Detection simplifies Angle Of Arrival (AOA) Processing.
    • Difference in Count along an antenna array provides Time Delay Of Arrival (TDOA)
  • Being a hardware solution, once validated, can be “Carved In Stone”. Whereas, with a Software Solution, there is always the possibility of problem-introduction for each new “Build/Release”.
  • TERAWIN is a hardware solution. Pulse DOPPLER and FMCW, being mainly software, require a lot more Computational Power and Storage, particularly FMCW (when using Ultra Wide Bandwidth with CHIRP Frame(s) and Multiple RX Antennas) for:
    • Target/Obstruction Detection.
    • Radial Speed Detection (DOPPLER).
    • Stationary Target Detection.
    • Constant False Alarm Rate (CFAR) Processing.
    • AOA for antenna arrays.
  • The TERAWIN Technique applies both to Pulse Radar (replaces software for DOPPLER, Target Detection and CFAR) and FMCW (replaces software for FFT, Target Detection and CFAR).
  • TERAWIN, and Associated Techniques, improve Mutual Interferences Rejection and Minimizes Mutual Interference Creation.
  • DynamicRangeTM Cell/Bin allows further minimization of Storage and Processing for Range Cells/Bins with no Target/Obstructions (SparseRangeTM); while applying patent pending VernierScaleTM “Magnification” for Range Cells/Bins with multiple Targets/Obstructions.
  • VernierScaleTM Processing also allows Dynamic (only when Detected)
  • Target/Obstruction IMAGING.
  • DuoPulseTM provides Target Velocity with just 2 reflections.
  • TrioPulseTM provides, with just 3 Reflections, Target Manoeuvring Detection for Interacting Multi-Model (IMM) Tracking:
    • Constant Velocity
    • Constant Acceleration (negative or positive)
    • Constant Turning
